Streaming Pipeline GCP

Pipeline en tiempo real con Pub/Sub y Dataflow para alimentar un feature store de Machine Learning con latencia inferior a 500 ms.

Descripción

El Streaming Pipeline GCP es un sistema de procesamiento de datos en tiempo real diseñado para manejar altos volúmenes de eventos con baja latencia. Utiliza Google Pub/Sub como sistema de mensajería distribuido para recibir eventos de múltiples fuentes de datos simultáneamente: APIs de clientes, sensores IoT y sistemas transaccionales.

El procesamiento se realiza con Apache Beam sobre Google Dataflow, lo que permite escalar automáticamente según el volumen de datos. Los pipelines incluyen transformaciones de enriquecimiento, validación de esquemas con Avro, detección de anomalías estadísticas y ventanas deslizantes de 5 y 60 minutos para agregaciones temporales.

Los datos procesados alimentan directamente el Feature Store ML, donde se almacenan y sirven features pre-calculadas para los modelos de Machine Learning en producción. Esto reduce significativamente la latencia de inferencia y garantiza consistencia entre training y serving (training-serving skew cercano a cero).

La arquitectura completa maneja más de 5 millones de eventos por hora con una latencia promedio inferior a 500 ms. El sistema está monitoreado con Cloud Monitoring, con alertas automatizadas configuradas para latencia y tasa de errores, y desplegado con Terraform para infraestructura como código.
